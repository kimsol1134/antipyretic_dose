# 마케팅 성장 전략 비판적 검토 프롬프트

**목적:** 작성된 마케팅 성장 전략을 다각도에서 비판적으로 분석하여 잠재적 약점, 과대평가된 가정, 실행 장애물을 식별하고 개선안을 제시합니다.

---

## 검토 지침

아래 프롬프트를 Claude, ChatGPT, Gemini 등 AI 모델 또는 실제 마케팅 전문가에게 제공하여 전략을 검증하세요.

---

## 비판적 검토 프롬프트

```
당신은 10년 이상의 경력을 가진 디지털 마케팅 컨설턴트이자 스타트업 성장 전략 전문가입니다.
특히 의료/헬스케어 분야 디지털 제품의 성장 전략을 다수 성공시킨 경험이 있습니다.

아래 마케팅 성장 전략 문서를 읽고, 다음 관점에서 냉철하게 비판적으로 검토해주세요.

---

### 1. 시장 및 경쟁 분석의 현실성

**검토 포인트:**
- 타겟 시장 규모 추정이 현실적인가?
- 경쟁 환경 분석이 충분한가? (기존 유사 서비스는 없는가?)
- 진입 장벽이 생각보다 낮아서 쉽게 복제될 위험은?
- "의사가 만든 도구"라는 차별화 요소가 실제로 경쟁 우위가 될까?

**비판적 질문:**
1. 한국/미국 시장에 이미 유사한 계산기가 존재하는가?
2. 제약회사(GSK, McNeil)가 자체 공식 도구를 제공하면 어떻게 대응할 것인가?
3. 네이버/구글이 Featured Snippet으로 직접 답변 제공 시 트래픽 감소 위험은?
4. 병원/약국 웹사이트에 이미 용량표가 있다면 계산기의 추가 가치는?

**평가 요청:**
- 시장 분석의 맹점 3가지 이상 지적
- 간과된 경쟁자 또는 대체재 제시
- 차별화 전략의 지속 가능성 평가

---

### 2. 트래픽 및 성장 가정의 타당성

**검토 포인트:**
- Phase 1에서 월 50,000 방문자 달성이 3개월 내 현실적인가?
- Phase 2에서 4배 성장(월 200,000) 가능성은?
- Phase 3에서 월 500,000 방문자는 과도한 낙관론이 아닌가?

**비판적 질문:**
1. SEO만으로 3개월에 50,000 방문자가 가능한 근거는? (실제 사례 있는가?)
2. 백링크 전략이 실제로 Domain Authority 상승으로 이어질까?
3. 인스타그램/유튜브 채널을 0부터 시작해서 의미 있는 트래픽을 만들 수 있을까?
4. 육아 분야는 계절성이 있는가? (출산 시즌, 감기 시즌 등)

**평가 요청:**
- 각 Phase의 트래픽 목표를 "보수적", "현실적", "낙관적" 시나리오로 재조정
- 가장 위험한 가정 3가지 식별
- 트래픽 증가가 기대에 못 미칠 경우 Plan B 제시

---

### 3. SEO 전략의 실효성

**검토 포인트:**
- 블로그 콘텐츠 20개 작성이 실제 검색 순위 상승으로 이어질까?
- 의료 정보는 YMYL(Your Money Your Life) 카테고리로 구글이 엄격하게 평가하는데 대응 전략은?
- 네이버 VIEW 탭 진입이 생각보다 어려운 건 아닌가?

**비판적 질문:**
1. E-A-T 신호가 충분한가? (의사 인증, 의료 기관 소속 명시 필요?)
2. 백링크를 "의사 100명에게 이메일"로 얻는 것이 현실적인가? (응답률 예상은?)
3. Featured Snippet을 타겟팅하지만, 실제 획득 난이도는? (기존 강자 제치기 가능?)
4. 콘텐츠 제작 예산 $300/월로 품질 높은 의료 콘텐츠 가능한가?

**평가 요청:**
- YMYL 대응 전략의 허점 지적
- SEO 타임라인의 현실성 재평가 (보통 6-12개월 소요)
- 백링크 획득 대안 전략 제시

---

### 4. 수익화 전략의 현실성

**검토 포인트:**
- 월 10,000 방문자에서 $30-150 수익 추정이 정확한가?
- 프리미엄 구독 모델($2.99/월)에 1-2%가 전환될 근거는?
- B2B 라이선싱이 실제로 작동할까? (병원/약국이 비용 지불 의향 있나?)

**비판적 질문:**
1. 무료 계산기를 쓰던 사용자가 $2.99/월을 낼 동기가 충분한가?
2. 프리미엄 기능(히스토리 저장, 알람)이 페이월을 정당화할 만큼 가치 있나?
3. 쿠팡/아마존 제휴 수익이 실제로 예상만큼 나올까? (클릭률, 전환율 가정 검증)
4. B2B 고객이 직접 개발하는 것보다 라이선스 구매를 선호할 이유는?
5. AdSense 승인 후 실제 RPM이 $1-5 범위에 들 근거는? (의료 니치 CPM은?)

**평가 요청:**
- 각 수익 채널의 "최악", "현실", "최선" 시나리오 재산정
- 수익 목표 달성 실패 시 대안 제시
- 프리미엄 기능 대신 다른 수익 모델 제안

---

### 5. 실행 가능성 및 리소스 제약

**검토 포인트:**
- 1인 개발자(solkim)가 모든 전략을 실행할 수 있을까?
- 주간 블로그 3개 + 유튜브 + 인스타그램 + 네이버 지식iN을 동시 운영 가능?
- 예산 $770/월이 충분한가? 과소평가된 비용은?

**비판적 질문:**
1. 콘텐츠 제작 시간이 얼마나 소요되나? (주당 몇 시간 필요?)
2. 의사가 본업이 있다면 부업으로 이 전략 실행 가능한가?
3. 외주 인력(작가, 디자이너) 관리 오버헤드는 고려되었나?
4. 법률 검토, 의료 감수 비용은 예산에 포함되었나?
5. 기술 유지보수(버그 수정, 보안 업데이트) 시간은?

**평가 요청:**
- 실행 우선순위 재조정 (80/20 법칙 적용)
- 1인 창업자가 실제로 집중해야 할 Top 3 활동 제시
- 숨겨진 비용 항목 식별

---

### 6. 리스크 관리의 충분성

**검토 포인트:**
- 5가지 리스크만 식별했는데, 간과된 리스크는 없나?
- 각 리스크의 완화 전략이 구체적이고 실행 가능한가?

**비판적 질문:**
1. **법적 리스크:** 잘못된 용량 정보로 인한 의료사고 발생 시 책임은? 면책 조항으로 충분한가?
2. **데이터 신뢰성:** products.json 데이터가 오래되면? 자동 업데이트 메커니즘은?
3. **플랫폼 리스크:** Vercel 요금 폭증, AdSense 계정 정지, 쿠팡 파트너스 계약 종료 시나리오는?
4. **평판 리스크:** 부정적 리뷰, 의료 커뮤니티 비판 대응 계획은?
5. **기술 부채:** 빠른 기능 추가로 인한 코드 품질 저하 위험은?

**평가 요청:**
- 치명적일 수 있는 리스크 Top 3 재평가
- 각 리스크의 발생 확률 × 영향도 매트릭스 작성
- 보험, 법률 자문 등 전문가 개입 필요 여부 판단

---

### 7. 타이밍 및 시장 진입 전략

**검토 포인트:**
- 지금이 최적의 진입 시점인가?
- 계절성을 고려한 마케팅 타이밍은?

**비판적 질문:**
1. 독감 시즌(겨울)에 론칭하는 것이 유리한가, 불리한가?
2. 경쟁자가 아직 없다면 시장 자체가 작은 건 아닌가?
3. 코로나 이후 원격의료/디지털 헬스케어 트렌드를 활용할 여지는?
4. 정부 정책 변화(온라인 의료정보 규제 강화)에 대한 대비는?

**평가 요청:**
- 시장 진입 타이밍의 기회/위협 분석
- 계절별 마케팅 캠페인 조정안 제시

---

### 8. 측정 및 최적화 전략

**검토 포인트:**
- KPI가 실제로 성공을 측정하기에 충분한가?
- A/B 테스트, 전환율 최적화 계획은?

**비판적 질문:**
1. "계산기 사용률 60%" 목표가 의미 있나? (사용 후 만족도는?)
2. Cohort 분석, Retention Rate 추적 계획은?
3. 어떤 지표가 개선되지 않으면 전략 피봇할 것인가? (Kill Criteria)
4. 실험 문화(A/B 테스트)가 1인 팀에서 가능한가?

**평가 요청:**
- North Star Metric 재정의 (가장 중요한 1개 지표)
- 조기 경보 지표 (Early Warning Indicators) 제시
- 데이터 기반 의사결정 프로세스 보완

---

### 9. 장기 지속 가능성

**검토 포인트:**
- 12개월 후에도 이 비즈니스가 성장할 수 있을까?
- Exit 전략 또는 장기 비전은?

**비판적 질문:**
1. 계산기 한 개로 지속 가능한 비즈니스가 될까? (제품 확장 필수?)
2. "육아 건강 플랫폼"으로 확장 시 경쟁 환경은? (기존 강자: 베이비센터, 앱스토어 육아앱들)
3. 투자 유치 가능성은? (VC 관심 받을 만한 시장 크기?)
4. Acqui-hire 또는 M&A 잠재 바이어는? (제약회사, 육아 플랫폼)

**평가 요청:**
- 3년 후 비전의 현실성 평가
- 지속 가능한 경쟁 우위 요소 식별
- Exit 옵션 평가 (유지 vs 매각 vs 투자 유치)

---

### 10. 전략적 대안 제시

**검토 포인트:**
- 제시된 전략 외에 더 나은 접근법은 없는가?

**비판적 질문:**
1. SEO 대신 유료 광고(Google Ads, 네이버 키워드)가 더 빠르지 않을까?
2. B2C 대신 B2B(병원 납품)에 집중하는 것이 수익성이 높지 않을까?
3. 앱 출시(iOS/Android)가 웹보다 retention이 좋지 않을까?
4. 인플루언서 마케팅에 예산을 집중하는 것이 효율적이지 않을까?
5. API 제공으로 다른 육아 앱에 통합되는 전략은?

**평가 요청:**
- 제시된 전략의 근본적 약점 3가지
- 완전히 다른 접근법(Contrarian Strategy) 제안
- "만약 예산이 10배라면?" / "만약 예산이 1/10이라면?" 시나리오별 전략

## 검토 결과 요청 형식

위 10개 관점에서 검토한 후, 다음 형식으로 답변해주세요:

### 1. 치명적 결함 (Critical Flaws)
- 즉시 수정해야 할 전략적 오류 3가지

### 2. 과대평가된 가정 (Overoptimistic Assumptions)
- 현실성 재평가가 필요한 가정 5가지

### 3. 간과된 리스크 (Blind Spots)
- 문서에서 다루지 않은 중요한 리스크 3가지

### 4. 실행 우선순위 재조정
- 80/20 법칙 적용 시 집중해야 할 Top 5 액션 아이템

### 5. 대안 전략 (Alternative Approaches)
- 더 효과적일 수 있는 3가지 다른 접근법

### 6. 예산 및 리소스 재평가
- 숨겨진 비용, 과소평가된 항목

### 7. 타임라인 현실화
- 각 Phase의 기간을 보수적으로 재조정

### 8. 최종 권장사항
- "이 전략을 실행한다면 반드시 수정/추가해야 할 3가지"

## 추가 검토 프롬프트 (특화 버전)

### A. 의료 전문가 관점 검토

```
당신은 소아과 전문의이자 디지털 헬스케어 스타트업 자문을 해본 경험이 있습니다.
의료 윤리, 환자 안전, 법적 책임 관점에서 이 전략을 검토해주세요.

특히 다음을 중점적으로:
1. 의료정보 제공의 법적 경계 (의료행위 vs 정보 제공)
2. 잘못된 정보로 인한 사고 발생 시 책임 소재
3. 의료 커뮤니티의 수용 가능성 (의사들이 추천할까?)
4. 환자 안전을 위해 추가해야 할 안전장치
```

### B. 육아 인플루언서 관점 검토

```
당신은 10만 팔로워를 가진 육아 인플루언서이자 맘카페 운영자입니다.
실제 부모들의 니즈와 행동 패턴 관점에서 이 전략을 검토해주세요.

특히:
1. 부모들이 실제로 이 도구를 쓸까? (기존 대안과 비교)
2. 바이럴 가능성이 있는 콘텐츠인가?
3. 맘카페에서 자연스럽게 공유될까, 광고로 받아들여질까?
4. 프리미엄 기능에 돈을 낼 부모가 있을까?
```

### C. 성장 마케터 관점 검토

```
당신은 SaaS/앱 스타트업을 0에서 월 100만 사용자로 성장시킨 경험이 있는 Growth Hacker입니다.
데이터 기반, 실험 중심, 확장 가능성 관점에서 이 전략을 검토해주세요.

특히:
1. Viral Loop, Network Effect 설계 가능성
2. Product-Market Fit 검증 계획의 적절성
3. Growth Hacking 기법 적용 가능 영역
4. Retention Strategy의 구체성
```

### D. 투자자 관점 검토

```
당신은 초기 단계 스타트업에 투자하는 VC 파트너입니다.
투자 매력도, 시장 규모, 수익성, Exit 가능성 관점에서 이 전략을 검토해주세요.

특히:
1. TAM, SAM, SOM 추정의 현실성
2. Unit Economics (CAC, LTV) 건전성
3. 확장 가능성 (Scalability)
4. Defensibility (모방 방지 능력)
5. Exit 시나리오 (M&A 바이어, IPO 가능성)
```

## 사용 방법

1. **Step 1:** 위 "비판적 검토 프롬프트" 전체를 복사
2. **Step 2:** `marketing-growth-strategy.md` 내용과 함께 AI에게 제공
